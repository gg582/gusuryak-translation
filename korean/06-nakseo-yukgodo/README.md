# 落書六觚圖 복원 탐색 프로젝트

주석 OCR의 수치 조건을 만족하는 배치(최적해)를 탐색으로 역산하는 Python 3 프로그램.

## 기하 구조 (확정)

《漢書·律曆志》 "二百七十一枚而成六觚, 爲一握" 및 蘇林 注 "其表六九五十四, 算中積凡得二百七十一枚"과
주석의 수치가 정확히 일치한다.

- 한 변 10칸의 육각 격자: 중심 1 + 고리 k(각 6k칸, k=1..9) = **271칸**
- **虛一**: 중심을 비움 → **270칸** (共積二百七十 / 虛一則二百七十數)
- 외주 **54칸** (校計周五十四數 = 蘇林注의 六九五十四)
- 총 칸 수 270 = 6×(1+2+...+9) = **6×45** (通加洛書數六倍)
- 중앙 가로줄(中觔) **19칸** (十九爲中觔數也)

## 가설 (최석정의 다른 마법진 수법과 동일)

- 값 1..270을 한 번씩 배치 (지수귀문도 1..30, 후책용구도 1..72 등과 같은 체계)
- 대점(점대칭) 위치의 두 값은 합 **271**의 보수쌍 (中上龜九圖 등의 보수쌍 수법)

이 가설 하에서 고리 k 합 = 813k, 축 합 = 2439는 구조적으로 자동 성립하고,
탐색은 변·섹터·광선의 균형만 찾는다.

## 실행

```bash
python3 tests/test_hexgrid.py     # 기하 불변량 테스트
python3 main.py                   # 탐색 → 도안 → 성질 분석 (output/)
python3 main.py --render-only     # 저장된 해로 그림·리포트만 재생성
```

## 탐색 결과 (output/ 기준)

**이론적 하한과 일치하는 최적해(페널티 6.0)를 찾았다.** 검증된 성질:

| 성질 | 목표 | 실측 |
|---|---|---|
| 대점쌍 합 | 271 | 135쌍 전부 정확 |
| 고리 k 합 | 813k (813, ..., 7317) | 9개 고리 전부 정확 |
| 외주 6변 합 | 각 1355 | 6변 전부 정확 |
| 6觚 섹터 합 | 6097/6098 | 6097,6098,6098,6098,6097,6097 |
| 6 광선 합 | 1219/1220 | 1219,1220,1219,1220,1219,1220 |
| 3축/中觔 합 | 각 2439 | 3축 전부 정확 |
| 꼭짓점 값 합 | 813 (=3×271, 구조적) | 206+126+245+65+145+26 = 813 |

섹터·광선 합은 칸 수가 홀수(45칸/9칸)라 정확한 균등이 불가능하며,
6097/6098·1219/1220 교대가 수학적 최선이다.

## 주의

이 배치는 주석의 승적법 절차를 그대로 재현한 것이 아니라, 주석의 **수치 조건을
만족하는 배치를 역산한 최적해**다.

## 가설 검증 결론 (`output/hypotheses.json`)

1. 주석에서 판독 가능한 수치는 전부 **칸 수 검산**이다 — 54+6=60, 60/6=10(변당 칸),
   中觔 19, 252(=271−19)×2=504, 270=6×45. 즉 주석의 검증 가능한 내용은
   도안의 기하 구조이며, 이는 전부 확인되었다.
2. 添六를 **값 배치 규칙**으로 읽는 건설 가설은 전부 실패한다:
   - ±6 (mod 271) 나선 배치 192변형: 마법 합 성질 불만족 (최선 벌점 27672)
   - 고리별 +6 등차 배치(모형 B): 고리별 시작값이 a_k = 274−18k로 유일하게
     존재해 고리 합 813k는 정확히 맞지만, 값이 고리 간에 중복(서로 다른 값 54개뿐)
     되어 1..270 배치 조건 불만족
3. 따라서 대점 보수쌍(합 271) 모형의 탐색 최적해가 현재 최선의 복원 후보이며,
   배치 순서 규칙(寄左/序左 구절)은 더 선명한 판본이 필요한 미해결 항목이다.
