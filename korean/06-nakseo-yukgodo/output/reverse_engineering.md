# 洛書六觚圖 — 최종 재구 도안의 생성 규칙 역산 및 주석 대조

재구된 최적해(`output/solution.json`)를 대상으로, 압축적인 생성 규칙이
남아 있는지를 데이터 주도로 검사하고, 결과를 흐린 주석의 판독 구절과
하나씩 대조한다.

## 1. 최종 재구 도안의 위치

- 탐색 목적 함수의 이론적 하한: 6.0 (달성).
- 다른 시드(42)로 찾은 최적해와의 일치 칸: **0/270**.
- 즉 조건을 만족하는 배치는 다수 존재하며, 재구된 도안은 그중 하나의
  표본이다. 역산은 이 표본에 건설적 흔적이 남는지를 묻는다.

## 2. 역산 시도와 결과

| 후보 규칙 | 방법 | 결과 |
|---|---|---|
| 고리 순회 등차 (임의 스텝 mod 271) | 고리별 270스텝 전수 | 실패 — 최선 일치율 16.7% |
| 좌표 선형 모형 v ≡ a+b·k+c·j | (a,b,c) 271³ 전수 | 실패 — 9/270칸 |

| 대점쌍 건설적 배정 순서 | 나선 순회 기준 연속 구간 | 없음 — 최장 2쌍 |
| 광선 차분 규칙 | 마주 보는 광선 비교 | 부호 반전 — 대점쌍 가설의 자동 귀결 |

## 4. 판정: 알고리즘 확정 가능 여부

알고리즘 확정 불가 (현재 증거로는). 기하 골격(271/270/54/19/252)과 합 조건(고리 813k·변 1355·축 2439·대점쌍 271)은 주석과 완전히 정합하지만, 값의 배치 순서 규칙은 재구된 어떤 최적해에도 압축적인 흔적으로 남아 있지 않다. 添六의 값 규칙 독해는 전부 반증되었고, 배치 순서 구절(寄左/序左)은 최적해가 다수 존재해 재구 도안만으로는 판정할 수 없다. 확정 가능한 것은 '무엇이 아닌가'의 범위까지이며, 알고리즘 본문의 확정에는 더 선명한 판본이 필요하다.

### 확정된 것

- 기하 골격: 271칸(虛一 → 270), 외주 54, 한 변 10칸, 中觚 19칸.
- 합 조건: 대점쌍 271, 고리 813k, 변 1355, 축 2439, 섹터 6097/6098,
  광선 1219/1220 — 주석의 수치와 《漢書》 六觚 기록에 정합.
- 添六·寄左 계열 구절은 값 규칙이 아니라 칸 수·순서 지시로 읽어야
  한다는 방향성.

### 확정되지 않은 것

- 값을 칸에 배정하는 절차(승적법 본문). 재구 최적해들 사이의 겹침이
  0/270이므로, 합 조걸만으로는 원래 배치를 특정할 수 없고, 원래 배치가
  따른 순서 규칙도 이 표본들에서 복구할 수 없다.
- 알고리즘의 확정에는 더 선명한 주석 판본이 필요하다.
