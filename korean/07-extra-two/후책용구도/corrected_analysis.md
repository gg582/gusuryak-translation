# 후책용구도 정정본 분석

이 문서는 원본 재구본과 숫자 완전 정정본의 차이를 정리한다. 원본
재구본 역시 리브레위키 후책용구도 서술의 규칙을 따르는 도상으로
본다. 다만 원본 재구본은 수치 조건을 끝까지 최적화한 판본이 아니라,
원문 도상의 형태와 국소 쌍합 표기를 우선 보존한 보수적인 재구성이다.
정정본은 같은 규칙을 숫자 조건까지 완전히 만족하도록 정리한 별도
판본이다.

## 1. 적용한 규칙

후책용구도의 핵심 규칙은 두 종류의 쌍합으로 요약된다.

```text
팔각형: 73 × 4 = 292
사각형: 74 + 72 = 146
```

따라서 3×3 작업 단위에서 변 쌍 36개를 모두 세면, 전체 72개 값의 합
`1 + ... + 72 = 2628`과 맞아야 한다. 이를 만족하려면 변 쌍의 목표합
분포는 다음처럼 균형을 이루어야 한다.

```text
73쌍 18개, 74쌍 9개, 72쌍 9개
18×73 + 9×74 + 9×72 = 2628
```

원본 재구본의 표시 합은 `73쌍 17개, 74쌍 11개, 72쌍 8개`로
합이 2631이 되므로, 1..72 전체합과 동시에 성립할 수 없다. 따라서
정정본에서는 후책용구도 규칙에 맞추어 목표합 분포를 `18/9/9`로
맞춘다.

## 2. 정정 원칙

정정본은 다음 조건을 동시에 만족하는 배치를 정수계획으로 찾은 것이다.

- 1..72를 정확히 한 번씩 사용한다.
- 각 변 쌍의 합은 `73`, `74`, `72` 중 하나다.
- 각 작업 팔각형의 네 변 쌍 합은 모두 292다.
- 기존 작업 데이터의 위치값을 최대한 많이 유지한다.

최적해는 72개 위치 중 61개를 그대로 유지한다.

| 위치 | 변 | 원본 재구본 | 정정본 | 목표합 |
|---|---|---:|---:|---:|
| 1행 1열 | left | (33,45) | (32,42) | 74 |
| 1행 2열 | bottom | (64,4) | (69,4) | 73 |
| 1행 3열 | top | (73,1) | (72,1) | 73 |
| 2행 1열 | top | (18,55) | (20,53) | 73 |

값 단위로는 11개 위치값이 달라진다. 쌍 단위로는 8개 변 쌍이 조정된다.

## 4. 정정본의 규칙 검증

정정본의 변 쌍 합 분포는 다음과 같다.

```text
72: 9개
73: 18개
74: 9개
```

각 작업 팔각형의 합은 모두 292다.

| 위치 | top+bottom | left+right | 전체 |
|---|---:|---:|---:|
| 1행 1열 | 146 | 146 | 292 |
| 1행 2열 | 146 | 146 | 292 |
| 1행 3열 | 146 | 146 | 292 |
| 2행 1열 | 146 | 146 | 292 |
| 2행 2열 | 146 | 146 | 292 |
| 2행 3열 | 146 | 146 | 292 |
| 3행 1열 | 146 | 146 | 292 |
| 3행 2열 | 146 | 146 | 292 |
| 3행 3열 | 146 | 146 | 292 |

또한 1..72가 정확히 한 번씩 사용된다.

## 5. 해석

정정본에서는 각 작업 팔각형이 두 개의 `73+73` 축과 하나의 `74+72`
축을 가진다. 그래서 각 단위는 항상

```text
(73 + 73) + (74 + 72) = 292
```

가 된다. 이는 후책용구도 설명에서 말하는 사선 변의 기준합 73과,
사각형을 만드는 양쪽 변의 `74/72` 보정이 같은 도상 안에서 함께
작동한다는 뜻이다.

따라서 corrected 판본은 단순히 합을 맞춘 임의 배열이 아니라,
리브레위키 후책용구도 서술의 생성 규칙을 3×3 작업 단위로 정리한
배치로 볼 수 있다.
